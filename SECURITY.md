# Security Policy

## Reporting Vulnerabilities

**DO NOT** open public GitHub issues for security vulnerabilities.

### How to Report

Email: **security@faultmaven.ai**

Include:
1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Affected versions (if known)
5. Suggested fix (if you have one)

### Response Timeline

| Stage | Timeframe |
|-------|-----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Resolution | Coordinated disclosure (typically 90 days) |

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x | ✅ |
| < 1.0 | ❌ |

## Security Best Practices

For self-hosted deployment security guidance, see:
[Security Guide](https://github.com/FaultMaven/faultmaven/blob/main/docs/SECURITY.md)

## Recognition

We appreciate responsible disclosure. Security researchers who report
valid vulnerabilities will be credited in our security advisories
(unless they prefer to remain anonymous).
